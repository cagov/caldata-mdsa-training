# GitHub Tutorials

Complete all five of the tutorials below. You can skip the <u>_What's next?_</u> section of each training.

1. Go through the [introduction to GitHub tutorial](https://github.com/skills/introduction-to-github)
    - You will learn how to create a branch, commit a change, open a pull request, and merge your change.

1. Learn how to [author GitHub-flavored Markdown](https://github.com/skills/communicate-using-markdown)
    - This involves no code. It is useful for staff to learn this material, including those who are not doing day-to-day coding, as it is used in issue tracking, cross referencing, and other project-management tooling.

1. Learn how to [review pull requests (PRs) on GitHub](https://github.com/skills/review-pull-requests)
    - You will learn how to open pull requests, review them, suggest changes, and merge them. Any change to the project codebase should go through this process, as it allows for peer review, automated checks and tests (CI), and automated deployments (CD).

1. Learn how to [resolve merge conflicts](https://github.com/skills/resolve-merge-conflicts)
    - Merge conflicts happen when two people make changes to the same file on GitHub. Learning how to resolve these conflicts is a good skill to practice.

1. Wrap up with [navigating your repository](https://github.com/skills/connect-the-dots)
    - This is useful for all staff, including those who are not doing day-to-day coding.
